# โฟลเดอร์และไฟล์
- public = โฟลเดอร์หลักสำหรับการติดต่อกับผู้ใช้
	- index.php = **ไฟล์หลักเพื่อรับการ request** จากผู้ใช้
	- .htaccess = ไฟล์ตั้งค่าระบบของ Apache web server
- includes = โฟลเดอร์ของไฟล์ระบบ
	- database.php = ไฟล์การ**เชื่อมต่อฐานข้อมูล**
	- router.php = ไฟล์**จัดการเส้นทาง**ของการ request
		- ใช้  file based routing คือ ชื่อเส้น จะเชื่อมไปยัง ชื่อไฟล์ใน routes
	- view.php = ไฟล์**จัดการการแสดงผล**หน้าต่างๆ
- routes = โฟลเดอร์ของการ**ประมวลผล**ตามลอจิคของแอปพลิเคชัน
	- home.php = ส่วนประมวลผลของหน้า Default
	- ping.php = ส่วนประมวลผลเมื่อมีการเรียกเส้นทางทดสอบ /ping
- templates = โฟลเดอร์การ**แสดงผล**ต่อผู้ใช้
	- 404.php = แสดงเมื่อผู้ใช้เรียกหน้าที่ไม่มีในแอปพลิเคชัน
	- home.php = ส่วนแสดงผลของหน้า Default

# การทำงาน
- ผู้ใช้ request เส้นทางเข้ามายังไฟล์ index.php
- ไฟล์ router.php จะตรวจสอบเส้นทางที่ request แล้วส่งต่อ (dispatch) ไปยัง routes ต่างๆ
- เช่นส่งต่อมาที่ home.php ก็จะประมวลผลตามลอจิคของแอปพลิเคชัน 
- เมื่อประมวลผลเสร็จและต้องแสดงผล จะส่งไปยัง template
- ไฟล์ที่กำหนดใน template เช่น home.php ก็จะแสดงผลลัพธ์ออกมา
